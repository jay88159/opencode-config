# 大纲示例：表达形态规划 + 图表清单

本文件提供 Step 1.3 的大纲示例与最小要求，核心目标是：在大纲阶段就规划“段落/分点/表/图”的节奏，避免正文变成纯文字墙。

## 最小要求（Checklist）

1. 4-6 章
2. 每章写清“要回答的问题”
3. 每章标注“表达形态”（段落/分点/表/图，可组合）
4. 只要计划出现列表，必须在大纲里先写 3-7 条“分点草案”（不是占位词）
5. 所有图表必须有唯一占位符 ID，并在大纲后输出图表清单

## 大纲格式示例

```markdown
## 1. 背景与目标
- 表达：段落（2 段）+ 分点列表（痛点 4 点）+ 【TBL-001：成功指标定义】
- 段落要回答：为什么现在做？不做会怎样？
- 分点列表草案（痛点）：
  - 搜索失败率高，用户需要多次重试
  - 结果排序不稳定，相关性弱
  - 缺少筛选/聚合，定位内容慢
  - 错别字/同义词召回差

## 2. 系统架构
- 表达：段落（1-2 段）+ 【FIG-001：系统架构图】+ 分点列表（组件职责 5 点）
- 分点列表草案（组件职责）：
  - Client：采集查询与上报埋点
  - API Gateway：鉴权、限流、路由
  - Search Service：召回、排序、聚合
  - Indexer：增量索引、回滚
  - Storage：索引存储与元数据

## 3. 详细设计
- 表达：段落 + 分点列表（流程分支）+ 【FIG-002：模块 A 流程图】+ 【TBL-002：接口参数说明】

## 4. 实施计划
- 表达：段落 + 【TBL-003：里程碑计划】+ 分点列表（风险 5 点）
```

## 图表清单示例

```markdown
### 图表清单

**图（共 2 个）**：
| ID | 名称 | 类型 | 所属章节 |
|----|------|------|----------|
| FIG-001 | 系统架构图 | 架构图 | 2. 系统架构 |
| FIG-002 | 模块 A 流程图 | 流程图 | 3. 详细设计 |

**表（共 3 个）**：
| ID | 名称 | 类型 | 所属章节 |
|----|------|------|----------|
| TBL-001 | 成功指标定义 | 指标表 | 1. 背景与目标 |
| TBL-002 | 接口参数说明 | 参数表 | 3. 详细设计 |
| TBL-003 | 里程碑计划 | 计划表 | 4. 实施计划 |
```

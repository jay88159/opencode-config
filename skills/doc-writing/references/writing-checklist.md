# 文档写作检查清单

在完成文档后，逐项核对以下检查清单确保质量。

---

## 一、结构检查

### 1.1 标题层级

- [ ] 全文只有一个 `#` 一级标题
- [ ] 层级递进合理（不跳级，如 `##` 直接到 `####`）
- [ ] 标题简洁准确，不超过 15 字
- [ ] 同级标题风格一致（并列结构）

### 1.2 章节完整性

- [ ] 有背景/目标说明（Why）
- [ ] 有核心内容（What/How）
- [ ] 有结论/后续步骤（So What）
- [ ] 章节之间有逻辑过渡

### 1.3 节奏与结构化

- [ ] 每个一级章节都明确了主要表达形态（段落 / 分点列表 / 表格 / 图，可组合）
- [ ] 凡是使用列表，列表前都有完整引导句（不是仅“如下”）
- [ ] 快速滚动浏览时，每 3-5 段能看到至少一个“休息点”（列表 / 表格 / 图 / 引用块）

---

## 二、段落检查

### 2.1 长度要求

- [ ] 每段 3-8 句（约 80-200 字）
- [ ] 无连续多个 1-2 句的碎段
- [ ] 短段仅用于过渡或强调

### 2.2 结构要求

- [ ] 每段有明确的主旨句
- [ ] 主旨句后有 2-5 句展开
- [ ] 使用了解释、举例、对比、因果等手法
- [ ] 段末有自然收束或过渡

### 2.3 列表要求

- [ ] 列表前有完整的引导句（非仅"如下："）
- [ ] 列表项风格统一（都是句子 or 都是短语）
- [ ] 有序列表用于有顺序的内容
- [ ] 无序列表用于并列项

---

## 三、用词检查

### 3.1 术语一致性

- [ ] 建立了术语表（首次出现时定义）
- [ ] 全文同一概念用同一词
- [ ] 缩写首次出现时给出全称

### 3.2 准确性

- [ ] 无模糊词（"大概"、"尽量"、"适当"）
- [ ] 数字、范围、优先级有具体值
- [ ] 专业术语定义准确

### 3.3 句式

- [ ] 一句一事，无超长复合句
- [ ] 因果、条件等逻辑关系显式标出
- [ ] 主被动态使用一致

---

## 四、图表检查

### 4.1 图

- [ ] 所有图都通过 draw-io 生成
- [ ] 每图有图题（图 X: [图题]）
- [ ] 图在正文中有引用（"如图 1 所示"）
- [ ] 图紧跟引用段落之后

### 4.2 表

- [ ] 每表有表题（表 X: [表题]）
- [ ] 表头清晰，字段名准确
- [ ] 表在正文中有引用
- [ ] 大表考虑拆分或加小标题

### 4.3 编号

- [ ] 图按出现顺序编号：图 1、图 2...
- [ ] 表按出现顺序编号：表 1、表 2...
- [ ] 编号全文连续或按章节编号（风格一致）

---

## 五、文风检查

### 5.1 正式度

- [ ] 无口语化表达
- [ ] 无网络梗或俚语
- [ ] 语气客观、克制

### 5.2 简洁性

- [ ] 删除了空转表述
- [ ] 无重复冗余内容
- [ ] 信息密度足够

### 5.3 时态

- [ ] 描述现状用现在时
- [ ] 描述规划用将来时
- [ ] 时态全文一致

---

## 六、写作之美检查（可能需要重写）

本步不仅检查格式，更要对文案进行**润色和重写**，确保文档具有节奏感、呼吸感和专业美感。

参考 `examples/good-vs-bad-paragraphs.md`。

### 6.1 六种平衡

- [ ] **长短平衡**：长短句交替，非全长句或全短句
- [ ] **虚实平衡**：概念有实例支撑，细节有观点提炼
- [ ] **密度平衡**：信息充实但有呼吸感，非堆砌或空洞
- [ ] **视觉平衡**：每 3-5 段有非文字元素（列表、表格、图）
- [ ] **结构平衡**：并列章节篇幅相近，非头重脚轻
- [ ] **语气平衡**：专业但有温度，非冰冷或随意

### 6.2 节奏感检查

- [ ] 大声朗读时有自然的起伏和停顿
- [ ] 非机械输出感（句式雷同、长度相近）
- [ ] 使用破折号、问句等变化句式

### 6.3 格式运用

- [ ] 关键术语、核心概念已**加粗**
- [ ] 术语首次定义时使用*斜体*
- [ ] 技术术语、命令、文件名用`代码格式`
- [ ] 格式使用有节奏，不单调也不过度

### 6.4 重写落地（强制）

- [ ] 发现碎段/空转/单调时，实际对原段落进行了重写/合并/拆分（而非只补图表描述）
- [ ] 重写后回归检查 2.1/2.2/2.3，确认段落质量与列表规范仍然满足要求
- [ ] 交付时附带“修订记录”（说明改了哪些段落/章节与原因）

---

## 七、排版检查

### 7.1 Markdown 语法

- [ ] 代码块指定了语言标记
- [ ] 表格对齐、列数一致
- [ ] 链接格式正确
- [ ] 无多余空行（连续空行不超过 2 行）

### 7.2 排版

- [ ] 中英文之间有空格
- [ ] 数字与中文之间有空格
- [ ] 标点符号使用正确（中文用中文标点）

---

## 八、保存检查

### 8.1 文件

- [ ] 文件保存到 `~/Documents`
- [ ] 文件名清晰（如 `PRD-搜索优化.md`）
- [ ] 使用 `.md` 扩展名

### 8.2 版本信息

- [ ] 文档顶部有版本号
- [ ] 有作者和日期
- [ ] 有状态标记（草稿/评审中/已定稿）

---

## 快速修复指南

| 问题 | 修复方法 |
|------|----------|
| 段落太短 | 合并相邻段落，或添加解释、举例 |
| 缺少主旨句 | 在段首添加概括性句子 |
| 术语不一致 | 全局搜索替换，统一用词 |
| 缺少图 | 使用 draw-io 生成 |
| 缺乏节奏感 | 重写段落，使长短句交替，加入变化句式 |
| 缺少格式强调 | 为关键词加粗，术语首次出现加斜体 |
| 缺乏呼吸感 | 每 3-5 段插入列表、表格或引用块 |
| 逻辑跳跃 | 添加过渡句，显式标出因果关系 |

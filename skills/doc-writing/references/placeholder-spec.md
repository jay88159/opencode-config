# 占位符规范：FIG/TBL

本文件定义 FIG/TBL 占位符的写法、描述深度、以及 Phase 3 的替换模板。

## 1. 命名与唯一性（强制）

- 图：`【FIG-001：图表名称】`
- 表：`【TBL-001：表格名称】`
- ID 必须唯一，且在全文保持不变（修订也不改 ID）

## 2. 占位符描述（写在正文里，Phase 2.1）

### 图占位符

```markdown
【FIG-001：系统架构图】
- 类型：架构图
- 包含组件：用户端、API 网关、微服务集群、数据库
- 展示关系：请求流向、服务间调用
- 重点标注：缓存层、消息队列
```

### 表占位符

```markdown
【TBL-001：接口参数说明】
- 列：参数名、类型、必填、说明、示例
- 行数：约 8-10 行
- 包含：userId、pageSize、sortBy 等参数
```

## 3. Phase 3 替换模板

### 图的替换

将 `【FIG-001：系统架构图】` 替换为：

```markdown
如图 1 所示，系统采用微服务架构...

![图 1 系统架构图](./assets/FIG-001-系统架构图.drawio.png)

*图 1 FIG-001 系统架构图*
```

### 表的替换

将 `【TBL-001：成功指标定义】` 替换为：

```markdown
表 1 列出了项目的核心成功指标。

*表 1 TBL-001 成功指标定义*

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| ... | ... | ... |
```

## 4. 融合校验（强制）

1. 搜索文档中是否还有 `【FIG-` 或 `【TBL-` 开头的占位符
2. 确保 `~/Documents/assets/` 中对应图表文件存在且路径正确

# 文风与结构化指南

本文件承载 doc-writing 的“写作细则”（更像手册/规范）。`SKILL.md` 只保留硬规则与阶段产物，避免过长。

## 1. 文风 (Writing Style)

向论文/高质量技术报告看齐：客观、一致、克制的正式感。

1. **客观克制**
   - 多用事实、数据、引用，少用主观揣测；结论要有依据。
   - 区分「既定事实」与「建议/判断」，必要时用「建议」「推荐」「在……前提下」标明。
2. **正式度一致**
   - 全文保持同一正式度：避免口语、网络梗、随意缩写（除非有统一术语表）。
   - 同一概念只用一种说法，不混用俗称、缩写、全称（若混用，首次出现时统一说明）。
3. **简洁去冗余**
   - 每段有明确主旨；删除与主旨无关的铺陈、重复句。
   - 避免空转表述（例如「在……的情况下」「从……的角度来说」）。
4. **时态与视角**
   - 描述现状用现在时；描述规划用将来时或「将/计划」。
   - 主被动尽量一致；优先主动态。

## 2. 段落与展开（禁止通篇碎段）

### 2.1 长度与密度

- **正文段落**：每段至少 3-8 句（或约 80-200 字），有完整观点展开
- **短段仅作例外**：强调、过渡、列表前的“完整引导句”
- **列表前的引导**：先写一句完整、有信息量的引导句，再列条目

### 2.2 段落结构

- **主旨句**：尽早点明本段讨论什么
- **发展**：用 2-5 句展开（解释/举例/对比/因果/条件/排歧义）
- **收束/过渡**（可选）：自然引向下段

### 2.3 自检

- 若某段只有 1-2 句：优先合并或扩写，除非它是刻意短段（过渡/强调）
- 若某节全是碎段：合并或重写，使每段有 3 句以上的实质展开

## 3. 文字与用词 (Wording)

1. **术语统一**
   - 建立并遵守术语表；缩写首次出现时给出全称
2. **准确、无歧义**
   - 避免模糊词（"大概"、"尽量"、"适当"）；范围/数量/优先级尽量量化
3. **句式与逻辑**
   - 一句一事；长句拆分；因果/条件/并列关系用连接词显式标出
4. **中英文混排**
   - 正文以中文为主；专有名词可保留英文；与图表/检索的术语保持一致

## 4. 图表搭配 (Figures and Tables)

1. **何时用图、何时用表**
   - 图：结构/流程/关系/状态/架构/时序
   - 表：对比/参数/字段/配置/清单
   - 一段话需要很多「其中/分别/如下」才能说清时，优先考虑图或表
2. **编号与题注**
   - 图题在图下方；表题在表上方；按出现顺序编号（风格统一）
3. **文中引用**
   - 正文必须显式引用每个图表（如图 X 所示、见表 X）
4. **分工**
   - 图/表讲“是什么/如何分布”，正文讲“为什么/怎么用/注意点”
